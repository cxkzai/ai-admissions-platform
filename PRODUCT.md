# Product

## Register

brand

## Users

**主要观众**（项目自身目的）
- 招聘方 / 面试官：30 秒内看懂"这个候选人做了什么"，进入演示台体验，留下一句"这人真有东西"
- 技术评审：能看到多智能体 + RAG + 工作流 + 全栈交付的真实代码

**项目最终用户**（产品目的）
- **教培机构招生主管**（管理层）：5-20 人团队管理者，要看实时漏斗、异常告警
- **一线课程顾问**（执行层）：年轻销售，靠提成吃饭，需要 AI 帮写话术、帮推荐课程、帮提醒跟进
- **机构老板**（决策层）：中小教培机构创始人，3-30 家分店，关注 ROI

**终端用户**
- 家长 / 学员（30-45 岁）：深夜咨询、希望得到专业且温暖的回应

## Product Purpose

**一句话**：让一个招生主管 + 一线顾问的团队效能翻倍，覆盖"线索获取 → 需求挖掘 → 试听转化 → 老带新裂变"招生全链路。

**项目本身的目的**：作为求职"AI 应用工程师"岗位的作品集，证明候选人在多智能体 / RAG 知识库 / 自动化工作流 / 全栈交付 4 个维度的真实能力。

**成功标准**：
- 30 秒内面试官能看懂项目做什么
- 演示台可立即点击体验 4 类 Agent
- RAG 引用、工具调用、对话管理都有可观测的实现
- README 完整 / 可一键启动 / 有真实部署 URL

## Brand Personality

**三个词**：**专业 / 温暖 / 极客**

- **专业**：教培行业的严谨感；每个细节都"是懂行的人做的"
- **温暖**：招生顾问服务的温度；不让 AI 显得冰冷机械
- **极客**：少儿编程 + AI 的专业感；用 IDE / 终端这种"程序员语言"作为视觉骨架

**情绪目标**：让观众感到"这产品真有人认真想过"，而不是"又一个套壳 AI"。

## Anti-references

**避免的视觉家族**（AI 默认套路）：
- ❌ Cream / sand / beige body bg（暖白底 + serif）——2026 年饱和的 AI 默认
- ❌ Dark + acid green / vermilion 单色高亮 ——Vercel/Linear 模仿
- ❌ Broadsheet-style（细线规、报纸列）——editorial 模仿

**避免的教培行业套路**：
- ❌ 编程猫 / 核桃编程的卡通风（彩虹色 + 卡通动物 + 低龄化视觉）
- ❌ 学而思 / 新东方的红色古典风（和家长群体强绑定，暗示"老牌"）
- ❌ 销售 SaaS 的冷淡工业风（Bootstrap admin 灰色密集感）

**避免的工程感**：
- ❌ 通用 Ant Design / shadcn/ui 默认外观（一眼能识别"套模板"）
- ❌ README 里贴技术栈列表当作价值证明（要演示不要罗列）

## Design Principles

1. **Practice what you preach** —— 我们做 AI 产品，前端自己就用 AI 友好的终端设计；每个元素都体现"AI 应用工程师"的审美
2. **Show, don't tell** —— 首页 Demo 即产品（演示台 30 秒可体验），技术栈列表不能取代真实演示
3. **Expert confidence** —— 不炫技，但每个细节都"是懂行的人做的"（如真实的 RAG 引用、真实的工具调用日志）
4. **Terminal-native identity** —— IDE 终端是少儿编程 + AI 工程师的共同语境，把"$" ">" "#" 提示符变成签名而非装饰
5. **Strategic restraint** —— 一处大胆（hero 配色 / 动效），其他地方安静克制；让观众一眼抓住记忆点

## Accessibility & Inclusion

**目标**：WCAG AA 级别

**具体要求**：
- 正文文字对比度 ≥ 4.5:1（已用深夜蓝 + 米白搭配达标）
- 大字体（≥ 18px 或加粗 ≥ 14px）对比度 ≥ 3:1
- 所有交互元素键盘可达
- 焦点状态可见（用 spark 暖橙 + 2px outline）
- 支持 `prefers-reduced-motion: reduce`（关闭非必要动画）
- 字号下限 14px，正文 16px
- 暗色主题为主（适合教培工作人员长时间使用）
- 屏幕阅读器友好（语义化 HTML、aria-label）
